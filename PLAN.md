# 故障机器人 Roguelike 幸存者 - 项目规划

## Context

以杀戮尖塔2的故障机器人（Defect/鸡煲）为主角，制作一个类 Vampire Survivors / Brotato 的 2D 俯视角 Roguelike 幸存者 Demo。核心特色：攻击卡牌循环系统 + 技能系统 + 充能球自动攻击。

## 技术选型

- **引擎**: Godot 4.6
- **视角**: 2D 俯视角 (Node2D)
- **语言**: GDScript

---

## 设计规则（Design Rules）

> 以下规则定义了 STS2 卡牌如何映射到本游戏系统中。修改这些规则会影响所有卡牌的实现方式。

### DR-1: 系统分类规则
- STS2 的 **Attack 卡** → 本游戏的 **自动攻击卡**
- STS2 的 **Skill 卡** → 本游戏的 **主动技能卡**
- STS2 的 **Power 卡** → 本游戏的 **被动能力卡**
- 充能球（Orb）→ **独立系统**，不归入以上三类
- 遗物系统

### DR-2: 攻击牌规则（实时抽牌出牌循环）
- 不再有独立的卡牌冷却，改为**能量限制**
- 每次抓取攻击牌时补充所有能量，视为进入新的一个**回合**
- 抓取后从左到右依次尝试释放所有手牌中的攻击牌
- 能量不足的牌跳过，直到所有攻击牌都被遍历
- 回合结束后所有攻击牌进入**弃牌堆**
- 从**抽牌堆**抓取新牌；抽牌堆不够时，将弃牌堆洗入抽牌堆（与 STS 原版相同）
- 每个回合有固定时间间隔（暂定 0.5s），可能通过其他系统改变
- 近战攻击逻辑：搜索近战范围内的随机一个敌人单位，对其所在一小块区域进行AoE攻击判定
- 投射物攻击逻辑：向攻击范围内最近的一个敌人发射一个投射物，投射物持续飞行，命中敌人后爆炸造成小范围效果
- 光束攻击逻辑：索敌逻辑与投射物类似（选择最近目标），发出一道射线，一小段时间内持续检测，对处于射线上的所有单位造成伤害与攻击效果
- 其他攻击的目标选取与弹道形式：按卡牌设定，见下方攻击卡数据表
- STS2 中的「消耗(Exhaust)」→ 转化为：待定
- STS2 中的「虚无(Ethereal)」→ 转化为：待定
- 回合中是否也能打出技能牌：待定
- 手牌上限：待定
- 每回合抓牌数量：待定

### DR-3: 主动技能卡规则
- 5 个技能槽，按键 1-5 主动释放
- 每个技能有独立冷却时间
- 充能补充机制：待定
- STS2 中的「费用」→ 转化为：待定
- STS2 中的「消耗(Exhaust)」→ 转化为：待定
- STS2 中的「虚无(Ethereal)」→ 转化为：待定

### DR-4: 被动能力卡规则
- 装备后永久生效
- 升级时作为奖励选项出现
- 叠加规则：待定

### DR-5: 充能球规则
- 环绕玩家旋转
- 球槽位默认数量：待定
- 被动触发频率：待定
- 激发(Evoke)触发条件：待定
- 最多同时存在的球数：待定

### DR-6: 通用转化规则
- STS2 的「格挡(Block)」→ 转化为：待定
- STS2 的「力量(Strength)」→ 转化为：待定
- STS2 的「聚焦(Focus)」→ 转化为：待定
- STS2 的「易伤(Vulnerable)」→ 转化为：待定
- STS2 的「虚弱(Weak)」→ 转化为：待定
- STS2 的「球槽(Orb Slot)」→ 转化为：待定
- STS2 的「引导(Channel)」→ 转化为：待定
- STS2 的「激发(Evoke)」→ 转化为：待定
- STS2 的「抽牌(Draw)」→ 转化为：待定
- STS2 的「弃牌堆(Discard Pile)」→ 转化为：待定
- STS2 的「状态牌(Status Card)」→ 转化为：待定
- STS2 的「回合制(Turn-based)」→ 转化为实时制，具体节奏：待定

## 核心系统架构

```
game/
├── main.tscn              # 主场景（游戏运行入口）
├── player/
│   ├── player.tscn        # 玩家场景（角色 + 碰撞体）
│   └── player.gd          # 玩家控制（移动、生命值）
├── cards/
│   ├── attack_card.gd     # 攻击卡牌基类
│   └── cards/             # 具体卡牌实现
├── orbs/
│   ├── orb.gd             # 充能球基类（环绕/自动攻击）
│   └── orbs/              # 具体充能球实现
├── skills/
│   ├── skill.gd           # 技能基类（冷却、释放）
│   └── skills/            # 具体技能实现
├── enemies/
│   ├── enemy.gd           # 敌人基类
│   ├── enemy_spawner.gd   # 敌人生成器
│   └── enemies/           # 具体敌人类型
├── xp/
│   ├── xp_gem.gd          # 经验宝石（掉落物）
│   └── level_manager.gd   # 升级管理（经验值、升级触发）
├── ui/
│   ├── hud.tscn           # HUD（血条、等级、技能栏）
│   ├── card_slots_ui.tscn # 攻击卡槽 UI
│   ├── skill_bar_ui.tscn  # 技能栏 UI
│   └── level_up_ui.tscn   # 升级选卡界面
└── wave/
    └── wave_manager.gd    # 波次管理（敌人生成节奏、难度递增）
```

## 系统概述（仅结构，不含转化后的数值）

### 1. 玩家系统
- 移动控制：WASD
- 基础属性：生命值、护甲、移动速度（具体数值待定）
- 球槽位（具体数值待定）

### 2. 自动攻击卡牌系统
- 5 个卡槽，卡牌循环自动释放
- 具体机制见 DR-2

### 3. 主动技能系统
- 5 个技能槽，按键 1-5 释放
- 具体机制见 DR-3

### 4. 充能球系统
- 独立系统
- 具体机制见 DR-5

### 5. 被动能力系统
- 升级奖励，永久生效
- 具体机制见 DR-4

### 6. 升级奖励系统
- 击杀敌人掉落经验宝石 → 自动拾取
- 经验满 → 升级 → 暂停 → 3 选 1
- 奖励池：自动攻击卡 / 主动技能 / 充能球 / 被动能力
- 槽位已满时的处理规则：待定

## 敌人与波次系统
- 敌人从屏幕外生成，向玩家移动
- 波次递增：数量↑、血量↑、新类型解锁
- 基础敌人类型：普通小兵、快速小兵、坦克、精英（具体设计待定）

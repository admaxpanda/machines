# 故障机器人 Roguelike 幸存者 - 项目规划

## Context

以杀戮尖塔2的故障机器人（Defect/鸡煲）为主角，制作一个类 Vampire Survivors / Brotato 的 2D 俯视角 Roguelike 幸存者 Demo。核心特色：攻击卡牌循环系统 + 技能系统 + 充能球自动攻击。

## 技术选型

- **引擎**: Godot 4.6
- **视角**: 2D 俯视角 (Node2D)
- **语言**: GDScript

---

## 设计规则（Design Rules）

> 以下规则定义了 STS2 卡牌如何映射到本游戏系统中。修改这些规则会影响所有卡牌的实现方式。

### DR-1: 系统分类规则
- STS2 的 **Attack 卡** → 本游戏的 **自动攻击卡**
- STS2 的 **Skill 卡** → 本游戏的 **主动技能卡**
- STS2 的 **Power 卡** → 本游戏的 **被动能力卡**
- 充能球（Orb）→ **独立系统**，不归入以上三类
- 遗物系统

### DR-2: 攻击牌规则（实时抽牌出牌循环）
- 不再有独立的卡牌冷却，改为**能量限制**
- 每次抓取攻击牌时恢复 3 点能量（默认值），视为进入新的一个**回合**
- 每回合从**抽牌堆**抓取 **5 张牌**；抽牌堆不够时，将弃牌堆洗入抽牌堆（与 STS 原版相同）
- **手牌上限为 10 张**；超过 10 张时不会进行多余抓取（相当于浪费抽牌机会）
- 回合开始后，每 **0.1s** 依次尝试打出一张手牌（从左到右）
- 能量不足的牌跳过，但**仍然消耗 0.1s**
- 打出的牌**先进入弃牌堆**，再进行抽牌
- 每个回合默认固定时间间隔为 **1s**，可能通过其他系统改变
- 若手牌全部判断完毕时总耗时未超过 1s，回合正常结束，等待剩余时间后进入下一回合
- 若总耗时超过 1s（手牌 > 10 张等极端情况），将下一回合延后，直到所有手牌都被判断完毕后才进入下一回合
- 近战攻击逻辑：搜索近战范围内的随机一个敌人单位，对其所在一小块区域进行AoE攻击判定
- 投射物攻击逻辑：向攻击范围内最近的一个敌人发射一个投射物，投射物持续飞行，命中敌人后爆炸造成小范围效果
- 光束攻击逻辑：索敌逻辑与投射物类似（选择最近目标），发出一道射线，一小段时间内持续检测，对处于射线上的所有单位造成伤害与攻击效果
- 其他攻击的目标选取与弹道形式：按卡牌设定，见下方攻击卡数据表

### DR-3: 主动技能卡规则
- 主动技能卡拥有**独立的卡池**（抽牌堆、弃牌堆、消耗堆），与攻击卡完全隔离
- 主动技能卡拥有**独立的能量系统**，与攻击卡能量互不影响
- 随每个攻击回合开始，从技能抽牌堆抽取 **1 张**技能卡，并获得 **1 点技能能量**；技能抽牌堆不够时，将弃牌堆洗入抽牌堆（与攻击卡规则相同）
- **技能能量没有上限**，可跨回合累积
- **手牌上限为 5 张**；超过 5 张时不再抽牌（浪费抽牌机会）
- 5 个技能槽位对应手牌位置，按 **1-5 键**主动释放对应位置的技能卡
- 释放技能卡消耗对应的技能能量（费用不足则无法释放）
- 技能卡打出后进入**技能弃牌堆**，洗牌后可再次抽到
- 技能卡的释放形式（弹道、范围、特效等）**逐卡单独定义**
- STS2 中的「费用」→ 技能卡能量消耗
- STS2 中的「消耗(Exhaust)」→ 单场战斗消耗：打出后移出当前战斗的技能牌池，下一场战斗重置牌池
- STS2 中的「虚无(Ethereal)」→ 5 个攻击回合后仍在手牌中的虚无牌进入消耗牌堆

### DR-4: 被动能力卡规则
- 装备后永久生效
- 升级时作为奖励选项出现
- 叠加规则：待定

### DR-5: 充能球规则
- 环绕玩家旋转
- 球槽位默认数量：待定
- 回合结束时依次执行每个充能球的**被动效果**
- 激发(Evoke)触发条件：被手动激发，或生成新充能球导致最老的充能球被挤出序列时激发
- 触发每个充能球（被动/激发）消耗 **0.02s**，插入在两个回合之间，对连续攻击回合进行中断
- 充能球类型：闪电、冰霜、黑暗、玻璃、等离子

### DR-6: Buff/Debuff 规则

> 从实现角度，力量、敏捷、集中、易伤、虚弱等均作为 **Buff** 统一实现，大部分被动能力卡的效果也通过 Buff 系统挂载。

- STS2 的「格挡(Block)」→ **临时护盾**：获得的护盾值持续 **5 个攻击回合**，未被消耗则消失
- **力量(Strength)**：每点力量使攻击牌伤害+1，不影响充能球
- **敏捷(Dexterity)**：每点敏捷在从攻击卡或技能卡获得格挡时额外+1，不影响充能球
- **集中(Focus)**：影响充能球被动/激发效果的数值，详见充能球章节
- **易伤(Vulnerable)**：受到攻击伤害增加25%
- **虚弱(Weak)**：造成的攻击伤害减少25%
- 易伤与虚弱可同时存在于同一单位
- 易伤/虚弱叠加只延长持续时间，不叠加效果倍率（如多次施加易伤仍是25%增伤，但持续时间累加）

## 核心系统架构

```
game/
├── main.tscn              # 主场景（游戏运行入口）
├── player/
│   ├── player.tscn        # 玩家场景（角色 + 碰撞体）
│   └── player.gd          # 玩家控制（移动、生命值）
├── cards/
│   ├── attack_card.gd     # 攻击卡牌基类
│   └── cards/             # 具体卡牌实现
├── orbs/
│   ├── orb.gd             # 充能球基类（环绕/自动攻击）
│   └── orbs/              # 具体充能球实现
├── skills/
│   ├── skill.gd           # 技能基类（冷却、释放）
│   └── skills/            # 具体技能实现
├── enemies/
│   ├── enemy.gd           # 敌人基类
│   ├── enemy_spawner.gd   # 敌人生成器
│   └── enemies/           # 具体敌人类型
├── xp/
│   ├── xp_gem.gd          # 经验宝石（掉落物）
│   └── level_manager.gd   # 升级管理（经验值、升级触发）
├── ui/
│   ├── hud.tscn           # HUD（血条、等级、技能栏）
│   ├── card_slots_ui.tscn # 攻击卡槽 UI
│   ├── skill_bar_ui.tscn  # 技能栏 UI
│   └── level_up_ui.tscn   # 升级选卡界面
└── wave/
    └── wave_manager.gd    # 波次管理（敌人生成节奏、难度递增）
```

## 系统概述（仅结构，不含转化后的数值）

### 1. 玩家系统
- 移动控制：WASD
- 基础属性：最大生命值 75，默认移动速度，无护甲设定
- 充能球槽位：默认 3 个

### 2. 自动攻击卡牌系统
- 5 个卡槽，卡牌循环自动释放
- 具体机制见 DR-2

### 3. 主动技能系统
- 5 个技能槽，按键 1-5 释放
- 具体机制见 DR-3

### 4. 充能球系统
- 独立系统
- 具体机制见 DR-5

### 5. 被动能力系统
- 升级奖励，永久生效
- 具体机制见 DR-4

### 6. 升级奖励系统
- 击杀敌人掉落经验宝石 → 自动拾取
- 在一场战斗结束后，结算提升的等级，每次提升获得一次选取能力的机会
- 奖励池：被动能力卡（每次升级可选一张）
- 每次战斗结束后可选取一张技能卡或攻击卡加入构筑

## 关卡与战斗系统

### 关卡结构
- 共 **3 个大场景**，每个场景 **5 关**，总计 **15 关**
- 每个关卡即视为一场「战斗」，战斗结束后结算升级并选卡
- 每关由多个波次构成，按时间逐步投放敌人

### 关卡难度分布（每个场景内）
| 关卡 | 敌人构成 |

| 第5关 | 普通怪物 + 关底生成 Boss |
